webpack2--loader的讲解

发表于 2019-03-13

webpack中的loader

webpack中里面有很多的loader和plugin，去官方文档里面可以看见几百个了.一般插件就是我们用啥就是使用啥.

在第一章中，我们大概了解到了模块化的概念，以及webpack的使用和webpack的配置文件的使用,这一章里面将为大家讲解一下webpack里面loader相关的概念.

file-loader

首先前面一章中我们大概已经了解到了怎么给js文件进行一个模块化的打包，可是如果现在我要去打包一张图片(jpg格式)，继续利用原来的方法去打包肯定是会报错的，于是这个时候，我们就需要引入一个叫做loader的东西了。

webpack.config.js

当然我们要记得安装一下这个loader

npm install file-loader -D

然后记得在index.js文件里面引入一下图片(因为打包的入口文件是index.js)

进行打包:

npm run bundle

我们用http-server起一波静态服务器

http-server -p 8081

访问localhost:8081/index.html
这里在控制台就会输出打包之后的图片的名字.

这里我们就用file-loader成功对图片进行了打包，它的工作原理就是遇到test里面所规定的文件类型，它会帮我们把这种文件类型移动到我们打包指定的文件夹里面去，然后返回这个文件的名称(当然打包之后的名称我们也可以自己定义)

其实file-loader可以帮我们处理很多其他类型的文件(例如txt,png)，它会有很多的处理情况，如果我们遇到了未知的处理情况，可以通过去官网查文档来解决这些问题的.

url-loader

url-loader不仅拥有很多file-loader的功能，而且还有很多新的功能

npm install url-loader -D

然后我们将之前配置文件中的file-loader更换成为url-loader,然后再去打包,这个时候我们却在dist目录下面看不见images目录了，但是当我们打开index.html的时候却能够看见我们引入的图片.

这个时候我们去dist的bundle.js文件里面去找，就会找到我们的图片转换的base64字符串,这样我们的图片就打包到了js里面.

但是这样使用并不是很正确的,它会导致我们的js文件变大(如果图片很大)，到时页面加载慢.

如果是小图片的话，我们是可以用可以通过url-loader将图片转换成base64编码放到js文件中去从而减少http请求的次数来优化性能.

url-loader其实和file-loader非常的类似，但是只是多了一个limit的配置

这里有一点很关键!!!

就是webpack里面的loader执行顺序是从下到上，从左到右。

上面的执行顺序就是先用sass-loader对样式文件进行打包，然后交给css-loader,css-loader打包完成之后，再给style-loader去打包挂载.

一般在页面中进行样式书写的时候，如果我们使用了关于css3里面的一些新的特性，一般是需要在样式前面加上一些前缀来支持不同的浏览器,比如-webkit-transform,但webpack里面有个loader可以让我们省去这些过程–postcss-loader

npm install postcss-loader -D

我们先根据官方文档对之进行一波安装,然后写一个配置文件

这样之后我们使用css3的新属性的时候，就会有插件帮我们自动去添加厂商前缀.
